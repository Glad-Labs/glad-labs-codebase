## ğŸš‚ Railway Deployment

When ready to deploy to Railway.app:

### 1. Connect your Git repository to Railway

- Push your code to GitHub
- Connect Railway to your repo

### 2. Add Environment Variables in Railway Dashboard

```

DATABASE_URL=[Auto-provided by Railway]
HOST=0.0.0.0
PORT=1337
APP_KEYS=<generate: openssl rand -base64 32>
API_TOKEN_SALT=<generate: openssl rand -base64 32>

TRANSFER_TOKEN_SALT=<generate: openssl rand -base64 32>
JWT_SECRET=<generate: openssl rand -base64 32>
```

### 3. Railway will automatically:

- Run: `npm install --omit=dev && npm run build`
- Start with: `npm run start`
- Restart on failure (up to 10 times)

## âš ï¸ Potential Issues & Solutions

### Issue: Port 1337 already in use

```bash
# Kill the process on that port (Windows)
netstat -ano | findstr :1337
taskkill /PID <PID> /F

# Or use a different port
PORT=1338 npm run dev
```

### Issue: SQLite database locked

```bash
# Delete the temp database and restart
rm -r .tmp
npm run dev
```

### Issue: Module not found errors

```bash
# Reinstall dependencies
rm -r node_modules
npm install
npm run dev
```
