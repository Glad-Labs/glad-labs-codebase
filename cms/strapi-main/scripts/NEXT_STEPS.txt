### üîµ PROGRAMMATIC (5 minutes) - Best for Automation

**Use API Token Registration - For CI/CD and Scripts**

This approach is best if you want automated/headless setup without admin panel.

**Steps**:

1. **Open Strapi Admin Panel**
   ```
   http://localhost:1337/admin
   ```

2. **Create Admin Account** (First time)
   - Email: `admin@gladlabs.com`
   - Password: `Admin@123456`
   - Click "Let's start"

3. **Create API Token**
   - Click Settings (gear icon ‚öôÔ∏è) in sidebar
   - Click "API Tokens"
   - Click "Create new API Token"
   - Fill in:
     - Name: `Content Type Registration`
     - Type: `Full access`
   - Click "Save"
   - **Copy the token immediately** (shown once only!)

4. **Set Token in PowerShell**
   ```powershell
   # In the same terminal where you'll run setup:
   $env:STRAPI_API_TOKEN = "paste-your-token-here"
   
   # Verify it's set:
   echo $env:STRAPI_API_TOKEN
   ```

5. **Run Registration**
   ```powershell
   npm run register-types:improved
   ```

## üÜò IF SOMETHING GOES WRONG

### Problem: Admin account creation fails

**Solution**:
1. Make sure Strapi is running: `npm run develop`
2. Wait 10 seconds for full startup
3. Clear browser cache: `Ctrl+Shift+Delete`
4. Try again: `http://localhost:1337/admin`

### Problem: Content types don't appear after admin creation

**Solution**:
1. Restart Strapi: Ctrl+C, then `npm run develop`
2. Wait 10 seconds
3. Refresh admin: F5
4. Check Content Manager again
5. If still missing: Run `npm run register-types:improved`

### Problem: API returns 404 for posts

**Solution**:
1. Confirm types in Content Manager (step 1)
2. Make sure types are registered (check admin)
3. Try restarting Strapi
4. Check for errors in Strapi logs

### Problem: Seed data fails

**Solution**:
1. First ensure ALL 7 types are registered
2. Check: http://localhost:1337/admin ‚Üí Content Manager
3. Should see all types listed
4. Try seed again: `npm run seed`
