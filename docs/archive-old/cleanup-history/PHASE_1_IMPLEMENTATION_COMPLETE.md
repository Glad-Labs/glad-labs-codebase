### 1. Documentation Framework (DOCUMENTATION_STRATEGY.md)

**5-Category System:**

1. **Architecture & Decisions** - Maintain actively (quarterly)
   - Core docs 00-07
   - WHY\_\*.md decision documents
   - Roadmaps

2. **Technical Reference** - Maintain actively (ongoing)
   - API contracts
   - Database schemas
   - Standards and guidelines

3. **How-To Guides** - Maintain minimally (as needed)
   - Stable, valuable topics only
   - Accept some staleness

4. **Troubleshooting** - Maintain as-needed (living document)
   - Common issues and solutions
   - Grow as problems solved

5. **Archive & History** - Never maintain (frozen)
   - Historical preservation
   - Reference only

**Philosophy:** "Pragmatism > Purity" - Document what helps, archive what clutters.
